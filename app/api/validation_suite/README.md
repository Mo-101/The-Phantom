# Phantom POE Gallabat-Metema Validation Suite

Purpose: turn the pre-registration plan into executable validation scaffolding without weakening the lock.

Doctrine: They add corridors. Phantom walks them. The harness proves the walking.

This bundle does **not** implement Phantom scoring logic. It calls the frozen engine through an adapter and enforces:
- temporal firewall
- manifest-only signals
- control corridor specificity
- append-only transition logs
- pre-declared metrics
- no reruns without a reason file

First lawful sequence:
1. Complete controls and state mapping.
2. Freeze engine commit/weights/thresholds.
3. Hash locked sections 1–6 of the pre-registration document.
4. Build/fill `signals_manifest.jsonl`.
5. Run replay exactly once.
6. Generate divergence report.

The uploaded v1.0 pre-registration plan remains the source of doctrine. This bundle is a builder scaffold, not a replacement.
